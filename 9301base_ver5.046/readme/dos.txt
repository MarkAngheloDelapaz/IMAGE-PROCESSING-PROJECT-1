ベースライブラリ Ver.5.000から
R_CSライブラリと一括で配布しています。

配布時のR_CSフォルダ構成は下記のとおりです。
	R_CSのインクルードファイル
		include/R_cs/
	R_CSのライブラリファイルはベースライブラリと同じフォルダにあります。
		dos/




ベースライブラリ配布時のディレクトリ構成が変更になっています。

2015年以前
　圧縮ファイルの中に「ヘッダファイル」「ライブラリファイル」「サンプルファイル」が存在していたため、
　同一ディレクトリに解凍して使用されていました。

2016年以降
　DOS版、Fedora11版(LinuxCUI用)、Fedora19版(LinuxGUI用)を一括管理するために、
　圧縮ファイルの中で、「ヘッダファイルディレクトリ」と「各システム用ディレクトリ」に分割されています。
　そのまま解凍すると、ヘッダファイルとライブラリファイルが異なるディレクトリになります。
　
2016年以降のライブラリ使用方法
　１）
　　2015年以前と同じディレクトリ構成で開発するためには、
　　ファイルを移動するなどの御対応をお願いいたします。
　２）
　　環境変数INCLUDEとLIBを変更すると、
　　そのままのディレクトリ構成で開発することが可能です。
　　例）
　　　2015年以前
　　　　SET INCLUDE=C:\WATCOM\H;C:\WATCOM\USER\TSC9301
　　　　SET LIB=C:\WATCOM\USER\TSC9301
　　　2016年以降
　　　　SET INCLUDE=C:\WATCOM\H;C:\WATCOM\USER\TSC9301\INCLUDE
　　　　SET LIB=C:\WATCOM\USER\TSC9301\DOS
